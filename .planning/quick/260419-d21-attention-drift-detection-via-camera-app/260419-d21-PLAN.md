# Plan: Attention Drift Detection (Camera + App)

## Scope

Add two drift-detection signals to glorb-timer that feed the existing `drift-detected` / `refocus-detected` IPC pipeline:

1. **App switch detection** — poll macOS active app every 2s; emit drift when not in LLM-determined whitelist.
2. **Camera presence detection** — hidden `getUserMedia` stream with a minimal motion/brightness heuristic; emit drift when no presence signal detected.

UI: add **task input** and **camera selection** controls in the existing timer window. On focus session start, compute whitelist (once) and start monitors. On timer end/cancel, stop monitors.

Scaled-down vs `../glorb` — no Python, no MediaPipe, no ADHD-aware scoring. Single heuristic per monitor.

## Tasks

### T1: Camera + app monitor modules (main process)

**Files:**
- `detect/ollama.js` — minimal Ollama JSON client (copy of `../glorb/app_detect/src/ollama.js`, trimmed)
- `detect/appMonitor.js` — osascript poll + whitelist classification

**Action:** Copy the Ollama client, port `getOpenApps`/`getActiveApp`/`predict` into a single module exporting `start({task, onDrift, onRefocus})` and `stop()`. Whitelist is computed once at start. Include hard-coded `alwaysAllowed` (Finder, System Settings, Glorb Timer, Electron).

**Verify:** `node -e "require('./detect/appMonitor')"` loads without error.

**Done:** Module exports `start`/`stop` and correctly maintains drift/refocus state by comparing active app to whitelist each tick.

### T2: Camera detection hidden window

**Files:**
- `detector.html` — minimal HTML with `<video>` element
- `detector.js` — `getUserMedia` + canvas frame sampler; emits `camera-drift` / `camera-refocus` via IPC

**Action:** Hidden BrowserWindow loads `detector.html`. Sample 320×240 frames at 2Hz. Compute mean brightness + frame-to-frame mean abs diff on a center ROI. "Presence" = brightness in [20, 240] AND any frame in last 15s had diff > 3. Emit state transitions only (edge-triggered).

**Verify:** Console logs presence/drift transitions when window is opened manually.

**Done:** Emits `camera-drift` IPC when presence lost >15s; emits `camera-refocus` when presence restored.

### T3: Main process wiring

**Files:** `main.js`, `preload.js`

**Action:**
- Add IPC handlers: `start-monitors`, `stop-monitors`, `list-cameras`, `camera-drift`, `camera-refocus`.
- `start-monitors({task, cameraDeviceId})` starts hidden detector window (passes `cameraDeviceId` via query string), starts app monitor, stores both in module-level refs.
- `stop-monitors` closes detector window and cancels app monitor polling.
- Camera/app monitors both call the existing drift/refocus code paths by calling the same logic as `drift-detected`/`refocus-detected` IPC handlers.
- Preload exposes `startMonitors`, `stopMonitors`, `listCameras`.

**Verify:** `grep startMonitors preload.js main.js` shows wiring; `npm start` boots without error.

**Done:** Timer Start in UI invokes `startMonitors`; Stop/complete invokes `stopMonitors`.

### T4: UI — task input + camera selection

**Files:** `renderer.html`, `renderer.css`

**Action:**
- Add a `<textarea id="task-input">` above the Start button (collapses when timer running).
- Add a `<select id="camera-select">` below it, populated via `listCameras()` on load.
- Expand the timer window from 286×468 → 286×560 to fit new controls.
- On Start click: call `startMonitors({task, cameraDeviceId})`. On reset/terminate: call `stopMonitors()`.

**Verify:** Manual: open app, type task, see camera list populate, click Start.

**Done:** Controls present, persist user's camera choice via `store`, call start/stop at timer boundaries.

## Must-haves

- Task input visible in timer window
- Camera selection dropdown populated from `navigator.mediaDevices.enumerateDevices()`
- `detect/appMonitor.js` with `start`/`stop` API
- `detect/ollama.js` with `classifyApps`
- Hidden detector window running camera presence heuristic
- Starting the timer starts both monitors; stopping/completing stops them
- Drift/refocus transitions call the same main-process code as existing `drift-detected`/`refocus-detected` handlers
