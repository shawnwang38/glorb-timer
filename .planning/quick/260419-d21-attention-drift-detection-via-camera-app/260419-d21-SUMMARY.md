# Summary

Added two drift-detection signals to glorb-timer that feed the existing
intervention pipeline (`triggerDrift` / `triggerRefocus` — same code paths
used by the Phase 8 `drift-detected` / `refocus-detected` IPC handlers and
the `/tmp/glorb-ipc.sock` CLI protocol).

## What changed

### App monitor (`detect/appMonitor.js` + `detect/ollama.js`)
- `osascript` polls active + open apps every 2s.
- On `start({task})`, queries Ollama `qwen3:1.7b` to classify which open
  apps are required for `task`; union with a small `alwaysAllowed` list
  (Finder, System Settings, Glorb Timer, Electron).
- Low-intent tasks (nothing / idle / break / relax / rest / killing time)
  skip the LLM and fall back to always-allowed only.
- If the LLM call fails, falls back to always-allowed (drift is
  still reported, just with a stricter baseline) — no hard crash.
- Edge-triggered: fires `onDrift` on the first tick the active app is
  off-whitelist, fires `onRefocus` on the first tick it returns.

### Camera monitor (`detector.html` + `detector.js`)
- Hidden 320×240 `BrowserWindow` loads `detector.html` with the selected
  `deviceId` passed via query string.
- `getUserMedia` → 160×120 canvas samples @ 2Hz.
- Presence heuristic: brightness in `[20,240]` AND at least one frame in
  the last 15s had mean-abs-diff > 3. When presence is lost for >15s,
  emits `camera-drift`; when restored, emits `camera-refocus`.
- Pure vanilla JS — no new npm deps, no ML models.

### Main process (`main.js`)
- Added `triggerDrift` / `triggerRefocus` helpers reused by the existing
  IPC handlers, the new camera detector IPC, and `appMonitor` callbacks
  (deduplicates three copies of the same logic).
- Added `createDetectorWindow(deviceId)` / `stopDetectorWindow()`.
- New IPC: `start-monitors({task, cameraDeviceId})`,
  `stop-monitors`, `camera-drift`, `camera-refocus`.
- Timer window height bumped 468 → 560 to fit new controls.

### Preload (`preload.js`)
- Main `window.glorb` bridge: added `startMonitors`, `stopMonitors`.
- New `window.glorbDetector` bridge (detector window only) with `drift`
  and `refocus`.

### Renderer (`renderer.html` + `renderer.css`)
- Task input (`<input id="task-input">`) and camera dropdown
  (`<select id="camera-select">`) above the Start button.
- Camera list primed via `getUserMedia` so labels populate on macOS
  (required — unprimed enumeration returns empty labels).
- Saved camera deviceId + last task text persist via `electron-store`.
- Start click → `startMonitors(task, deviceId)` + disables controls.
- `resetTimer()` (manual cancel, auto-complete, intervention terminate)
  → `stopMonitors()` + re-enables controls. Idempotent.
- Settings panel open/close resize calls updated for the new 560 height.

### Build config (`package.json`)
- Added `detector.html`, `detector.js`, `detect/**/*.js` to the
  electron-builder file list so the packaged DMG includes them.

## Not done (intentional — minimal scope)

- No head-pose scoring (original glorb used MediaPipe FaceMesh + yaw/
  pitch). Replaced with motion+brightness presence heuristic.
- No fatigue detection (EAR/BPM).
- No ADHD-aware tuning on the detection side — existing intervention
  pipeline already branches on `hasADHD` and is unchanged.
- No UI for the whitelist — it is computed once at timer start and
  logged to stdout only. If the user wants to inspect/edit it, that
  is a follow-up task.
- LLM model is hardcoded (`qwen3:1.7b`). If Ollama isn't running,
  only the always-allowed list is used.
